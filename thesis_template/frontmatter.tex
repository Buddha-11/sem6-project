\begin{abstract}
\begin{spacing}{2}

\addchaptertocentry{\abstractname}
Static Application Security Testing (SAST) tools such as CodeQL are widely employed in
modern CI/CD pipelines to detect vulnerabilities in source code. However, their practical
value is severely limited by excessive false positive alerts---studies report that
30--60\% of SAST alerts are false positives, causing alert fatigue and leading developers
to ignore these tools. This report presents the design, implementation, and evaluation of
an \emph{Agentic AI Pipeline for Automated Vulnerability Detection and Triage}, which
combines CodeQL static analysis with machine learning (ML)-based alert classification and
an agentic AI layer for autonomous vulnerability prioritisation and remediation.

Using the OWASP Benchmark dataset (2,766 Java test cases), we evaluate three classifiers:
Random Forest, Gradient Boosting, and Support Vector Machine. A vectorized 13-dimension
feature representation coupled with threshold optimization yields a Gradient Boosting
model achieving F1-score of 0.8886 and ROC-AUC of 0.9106. The agentic triage layer
converts high-confidence alerts into developer-actionable patches via LLM reasoning,
closing the loop from static analysis to automated remediation.

\end{spacing}
\end{abstract}
